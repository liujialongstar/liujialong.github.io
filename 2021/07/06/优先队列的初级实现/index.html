优先队列的初级实现

发表于 2021-07-06 分类于 算法

许多应用程序都需要处理有序的元素, 但不一定要求他们全部有序, 或是不一定一次就将它们排序. 很多情况下, 我们会收集一些元素, 处理当前最大的元素, 然后在收集更多的元素, 再处理当前键值最大的元素, 如此这般. 例如, 你可能有一台能够同时运行多个应用程序的电脑. 这是通过为每个应用程序的事件非配一个优先级, 并总是处理下一个优先级更高的事件来实现的.

这种情况下, 一个合适的数据结构应该支持两种操作: 删除最大元素和插入元素. 这种数据结构叫做优先队列.

优先队列的一些重要的应用场景包括:

  • 模拟系统, 其中事件的键即为发生时间, 而系统需要按照时间顺序处理所有事件;
  • 任务调度, 其中键值对应的优先级决定了应该首先执行哪些任务;
  • 数值计算, 键值代表计算错误, 而我们需要按照键值制定的顺序来修正它们.

下面, 我们来一步步实现优先队列.

API

优先队列作为一种抽象数据类型(ADT), 它表示了一组值和对这些值的操作, 他的抽象层是我们能够方便地将应用程序和各种具体实现隔离开来. 首先我们将定义一组API来表示优先队列.

优先队列最重要的操作就是删除最大元素和插入元素. 删除最大元素的方法名为delMax(), 插入元素的法那个法名为insert(). 为了将API定义完整, 我们加入了构造函数和一个空队列测试方法. 并且实现了泛型. 具体API列表如下表:

public class MaxPriorityQueue<T exendss Comparable<T>>
MaxPriorityQueue() 创建一个优先队列
MaxPriorityQueue(int max) 创建一个最大容量为max的优先队列
MaxPriorityQueue(T[] a) 用a[]中的元素创建一个与优先队列
void insert(T v) 向优先队列插入一个元素
T max() 返回最大元素
T delMax() 删除并返回最大元素
boolean isEmpty() 返回队列是否为空
int size() 返回优先队列中元素的个数

类似的, 可以创建一个MinPriorityQueue, 它使用delMin()方法来删除并返回队列中键值最小的那个元素.

使用无序序列是解决这个问题的惰性方法, 我们仅在需要的时候才采取行动(找出最大(小)元素); 使用有序序列则是解决问题的积极方法, 在插入元素的时候保持列表有序, 是后续操作更高效.

优先队列的初级实现效率并不高, 最坏情况下, 有序数组的插入操作和无需数组的删除操作都需要线性时间完成.
