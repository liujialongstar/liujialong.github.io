符号表(字典)的基本实现

发表于 2021-07-08 更新于 2021-07-20 分类于 算法

符号表是一种存储键值对的数据结构, 支持两种操作: 插入(put), 即将一组新的键值对存入表中; 查找(get), 即根据给定的键得到相应的值.

符号表有时被称为字典, 在英语字典中, 键就是单词, 值就是单词对应的定义, 发音和词源.

符号表有时又叫做索引, 即书本最后将术语按照字母顺序列出方便查找的那部分. 在一本书的索引中, 键就是属于, 而值就是书中该术语出现的所有页码.

接下来, 我们将实现两种简单的符号表.

API

符号表是一种典型的抽象数据结构: 它代表着一组定义清晰的值以及相应的操作, 使得我们能够将类型的实现和使用区分开来. 下面是符号表的API定义:

public class SymbolTable<K, V> 说明
SymbolTable() 创建一张符号表
void put(K key, V val) 将键值对存放再表中(若值为空则将键key从表中删除)
V get(K key) 获取键key对应的值(若键key不存在则返回null)
void delete(K key) 从表中删除键key及其对应的值
boolean contains(K key) 键key再表中是否存在对应的值
boolean isEmpty() 表是否为空
int size() 表中键值对的数量
Iterable keys() 表中所有键的集合

对于这份API, 使用了下面的这些设计决策.

泛型

设计方法时没有指定处理对象的类型, 而是使用了泛型. 对于符号表, 通过明确地指定查找时键和值地类型来区分它们地不同角色.

重复的键

所有的实现都遵循以下规则:

  • 每个键只对应一个值(表中不允许存在重复的键);
  • 当用例代码向表中存入的键值对和表中已有的键值对冲突时, 新的值会替代旧的值.

这些规则定义了关联数组的抽象形式. 可以将符号表想象成一个数组, 键即索引, 值即数组的元素. 在一个一般的数组中, 键就是整型的索引, 通过它来快速访问数组的元素; 再一个关联数组(符号表)中, 键可以是任意类型, 但仍可以用它来快速访问数组的内容.

空(null)键

键不能为空. 因为我们会用key调用compareTo()或equals()方法, null调用这两个方法会抛出一个空指针异常.

空(null)值

我们规定不允许有空值. 因为再我们的API定义中, 当键不存在的时候get()方法会返回空(null), 也就是说任何不再表中的键关联的值都是空. 这个规定会产生两个结果:

  1. 我们可以用get方法是否返回空来测试给定的键是否存在于符号表中;
  2. 我们可以将空值作为put()方法的第二个参数存入表中来实现删除;

删除操作

在符号表中, 删除的实现可以有两种方法:

  • 延时删除: 将键对应的值置为空, 然后在某个时候删去所有值为空的键;
  • 即时删除: 立刻从表中删除指定的键.

迭代

为了方便用例处理表中的所有键值, 我们有时会在API的第一行加上implements Iterable<K>, 强制所有实现都必须包含iterator()方法来返回一个实现了hasNext()和next()方法的迭代器. 但是对于符号表, 我们采用一个更简单的方法, 来返回一个Iterable<K>对象以方便用例遍历所有的键.

键的等价性

要确定一个给定的键是否存在于符号表中, 首先要确立对象等价性的概念. 在Java中, 所有的对象都继承了equals()方法. Java也为标准数据类型如String实现了equals()方法. 而自定义的键则需要重写equals()方法. 最好使用不可变的数据类型作为键, 否则表的一致性无法保证.

无序链表中的顺序查找

符号表中使用的数据结构的一个简单选择是链表, 每个节点存储一个键值对. get()的实现即为遍历链表, 使用equals()方法比较需要被查找的键和每个节点中的键, 如果匹配成功返回相应的值, 否则返回null. put()的实现也是遍历链表, 使用equals()方法比较需要被查找的键和每个节点中的键, 如果匹配成功, 就用第二个参数指定的值更新该键相关联的值, 否则就用给定的键值对创建一个新的节点并插入到链表的开头. 这种方法被称为顺序查找: 在查找中, 一个一个地顺序遍历符号表中地所有键并使用equals()方法来寻找与被查找地键匹配地键.

因此, 基于链表的实现以及顺序查找是非常低效的, 无法处理庞大的输入.

有序符号表

典型的应用程序中, 键都是Comparable的对象, 因此使用a.compareTo(b)来比较a和b两个键. 许多符号表的实现都利用了comparable接口带来的键的有序性来更好地实现put()和get()方法. 更重要的是在这些视线中, 我们可以认为符号表都会保持键的有序并大大扩展它的API.

有序数组的二分查找

有序符号表的实现, 使用了一组平行的数组作为数据结构, 一个存储键, 一个存储值. 该实现可以保证数组中的Comparable类型的键有序, 然后使用数组的索引来高效的实现get()和其他操作.

这份实现的核心时rank()方法, 他返回表中小于给定键的键的数量. 对于get()方法, 只要给定的键存在于表中, rank()方法就能够精确的告诉我们在哪里能够找到它(如果找不到, 那它肯定不在表中).

对于put()方法, 只要给定的键存在于表中, rank()方法就能够精确地告诉我们到哪里去更新它的值, 以及当键不存在于表中时, 将键存储到表的何处. 我们将所有更大的键向后移动一格来腾出位置, 并将给定的键值对分别插入到各自数组中的合适位置.
