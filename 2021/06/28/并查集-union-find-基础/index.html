动态连通性问题

对于一系列整数对, 其中每个整数都表示一个某种类型的对象, 一对整数p q可以被理解为”p和q是相连的”. 假设”相连”是一种对等的关系, 这就意味着它具有以下性质:

  • 自反性: p和p是相连的;
  • 对称性: 如果q和q是相连的, 那么q和p也是相连的;
  • 传递性: 如果p和q是相连的且q和r是相连的, 那么p和r也是相连的.

对等关系能够将对象分为多个等价类. 在这里, 当且仅当两个对象相连时, 它们才属于同一个等价类. 我们的目标是编写一个程序来过滤掉序列中所有相连的整数对(两个整数均来自于同一个等价类). 为了达到锁期望的效果, 我们需要设计一个数据结构来保存程序已知的所有整数对的足够多的信息, 并用它们来判断一堆新对象是否是相连的. 我们将这个问题成为动态连通性问题.

本文中将使用网络方面的术语, 将对象称为触点, 将整数对成为连接, 将等价类成为连通分量或简称为分量. 假设我们有用0到N-1的整数所表示的N个触点.

为了说明问题, 我们设计了一份API来封装所需的基本操作:

  • 初始化;
  • 连接两个触点;
  • 判断包含某个触点的分量;
  • 判断两个触点是否存在于同一个分量之中;
  • 返回所有分量的数量.

详细API如下表:

public interface UF 说明
UF(int N) 以整数标识(0到N-1)初始化N个触点
void union(int p, int q) 在p和q之间添加一条连接
int find(int p) p所在的分量的标识符(0到N-1)
boolean isConnected(int p, int q) 如果p和q存在于同一个分量中则返回true
boolean count() 连通分量的数量

如果两个触点在不同的分量中, union()操作会将两个分量归并. find()操作会返回给定触点所在的连通分量的标识符. isConnected()操作能够判断两个触点是否存在于同一个分量之中. count()方法会返回所有连通分量的数量, 一开始我们有N个分量, 将两个分量对并的每次union操作都会使分量总数减一.

为解决动态连通性问题设计算法的任务转化为了实现这份API. 所有实现都应该:

  • 定义一种数据结构标识已知的连接;
  • 基于此数据结构实现高效的union(), find(), isConnected()和count()方法.

API已经说明触点和分量都会用int值标识, 所以我们可以使用一个以触点为索引的的数组id[]作为基本数据结构标识所有分量. 我们将使用分量中的某个触点的名称作为分量的标识符, 因此可以认为每个分量都是由它的触点之一所表示的.

一开始, 我们有N个分量, 每个触点都构成了一个只含有它自己的分量, 因此我们将id[i]的值初始化为i, 其中i在0到N-1之间. 对于每个触点i, 我们将find方法用来判断它所在的分量所需的信息保存在id[i]之中. isConnected()方法的实现只需要用一条语句find(p) == find(q), 它返回一个布尔值, 我们所有方法的实现都会用到isConnected()方法.

接下来, 我们将讨论三种不同的实现

quick-find算法

一种方法是保证当且仅当id[p]等于id[q]时p和q是相连的. 即在同一个连通分量中, 所有触点在id[]中的值必须全部相同.

这意味isConnected方法只需要判断id[p] == id[q], 当且仅当p和q在同一分量中, 才会返回true.

为了保证这一点, 在调用union()方法是, 首先应该检查p和q是否已经在同一分量中, 如果是, 则不采用任何操作; 否则就将p所在分量中的所有触点的id[]值替换为q所在的分量值.

quick-union算法

另一种算法的重点是提高union()方法的速度, 它和quick-find算法是互补的, 并且是基于相同的数据结构–以触点作为索引的id[]数组. 但是我们赋予这些值的意义不同, 我们需要它们来定义更加复杂的结构. 每个触点所对应的id[]元素都是同一个分量中另一个触点的名称(或者它自己)–我们成这种联系为链接(类似链表, 每个触点保存它父触点的名称, 即父触点在id[]中的下表).

实现find() 方法是, 我们从给定的触点开始, 有它的链接得到另一个触点, 再由这个触点的链接达到第三个触点, 知道到达一个根触点, 即指向自己的触点. 当且仅当两个触点的根触点为同一个触点时, 它们存在于同一个分量中.

我们需要调用union()方法来保证这一点. 我们由p和q分别找到它们的根触点, 然后将要给根触点连接到另一个即可将一个分量重命名为另一个分量. 因此这个算法叫做quick-union.

本质上, quick-union实现了一种树了. 一颗树的大小是它的节点的数量. 树中的一个节点的深度是它到根节点路径上的链接数. 树的高度是它的所有节点中的最大深度.

quick-union算法仍存在一定的问题, 在极端情况下, 树的高度为N-1(每个节点最多只有一个子节点), 此时处理N对整数所需的所有find()操作访问数组的总次数为2*(1+2+…+N)~N^2, 这种情况下运行时间仍然是平方级别的.

加权quick-union算法

与其在union()中随意将一棵树连接到另一颗树, 我们现在会记录每一颗树的大小并总将较小的树连接到较大的树上. 这项改动需要添加一个数组和一些代码来记录树中的节点数. 我们称之为加权quick-union算法.
