散列表

发表于 2021-08-11 更新于 2021-08-12 分类于 算法

如果所有的键都是小的整数, 就可以使用一个数组来实现无序的符号表, 将键作为数组的索引而数组中键i处存储的就是它对应的值. 这样就可以快速访问任意键的值.

散列表就是这种简易方法的扩展并能处理更加复杂的类型的键.

散列表的查找算法分为两步. 首先使用散列函数将被查找的键转化为数组的一个索引. 理想情况下, 不同的键都能转化为不同的索引值. 但实际上需面对两个或多个键都散列到相同索引的情况. 因此, 散列查找的第二步就是处理碰撞冲突. 解决碰撞冲突主要使用拉链法和线性探测法.

散列表是算法再时间和空间上做出权衡的经典例子. 如果没有内存限制, 可以直接使用键作为(可能是一个超大的)数组索引, 那么所有查找操作只需要访问内存一次即可; 如果没有时间限制, 可以使用无序数组并顺序查找, 这样需要很少内存, 但时间是线性的.

而使用散列表, 就可以实现在一般应用中拥有常数级别的查找和插入操作的符号表.

符号表的二叉查找树实现

发表于 2021-07-20 更新于 2021-07-22 分类于 算法

一棵二叉查找树是一颗二叉树, 其中每个结点都含有一个Comparable的键(以及相关联的值), 且每个结点的键都大于其左子树中的任意结点的键而小于右子树的任意几点的键.

二叉查找树由结点组成, 结点包含的链接可以指向空(null)或其他结点. 在二叉树中, 每个结点只能有一个父结点指向自己(根结点除外, 它没有父结点), 而且每个结点都只有左右两个链接, 分别指向自己的左子结点和右子结点.

二叉树的链接指向的是结点, 但是我们可以将每个链接看作指向了另一棵二叉树, 而这棵树的根节点就是被指向的结点. 因此我们可以将二叉树定义为一个空链接, 或者是一个由左右两个链接的结点, 每个链接指向一个独立的子二叉树.

二叉查找树中, 每个结点还包含一个键和一个值, 键之间也是有顺序的, 可以支持高效的查找.

使用二叉查找树, 我们就可以将链表的灵活性和有序数组查找的高效性结合起来, 高效地实现符号表.

优先队列的堆实现

发表于 2021-07-06 更新于 2021-07-07 分类于 算法

堆是一类特殊的数据结构的统称. 堆通常是一个可以被看作一棵树的数组对象.

二叉堆是一种特殊的堆, 是一组能够用堆有序的完全二叉树排序的元素, 并在数组中按照层级存储(不使用数组的第一个位置). 二叉堆分为两种: 最大堆和最小堆. 最大堆父节点的键值总是大于或等于任何一个子节点的键值; 最小堆父节点的键值总是小于或等于任何一个子节点的键值.

完全二叉树的具体表示法是将二叉树的节点按照层级顺序放入数组中, 根节点在位置1, 它的子节点在位置2和3, 而子节点的子节点分别在位置4567, 以此类推.

在二叉堆中, 位置k的节点的父节点的位置为k/2, 它的子节点的位置分别为2k和2k+1. 这样在不使用指针的情况下, 也可以通过计算数组的索引在树中上下移动: 从a[k]向上一层就令k等于k/2, 向下一层则令k等于2k或2k+1.

使用二叉堆可以让我们高效地实现优先队列. 接下来我们将用二叉堆实现对数级别的插入元素和删除最小元素的操作. 一棵大小为N的的完全二叉树的高度为lgN, 利用这一点, 即可保证对数复杂度的性能.

优先队列的初级实现

许多应用程序都需要处理有序的元素, 但不一定要求他们全部有序, 或是不一定一次就将它们排序. 很多情况下, 我们会收集一些元素, 处理当前最大的元素, 然后在收集更多的元素, 再处理当前键值最大的元素, 如此这般. 例如, 你可能有一台能够同时运行多个应用程序的电脑. 这是通过为每个应用程序的事件非配一个优先级, 并总是处理下一个优先级更高的事件来实现的.

这种情况下, 一个合适的数据结构应该支持两种操作: 删除最大元素和插入元素. 这种数据结构叫做优先队列.

优先队列的一些重要的应用场景包括:

  • 模拟系统, 其中事件的键即为发生时间, 而系统需要按照时间顺序处理所有事件;
  • 任务调度, 其中键值对应的优先级决定了应该首先执行哪些任务;
  • 数值计算, 键值代表计算错误, 而我们需要按照键值制定的顺序来修正它们.

快速排序

发表于 2021-07-01 分类于 算法

快速排序是一种分治的排序算法. 它将一个数组分成两个子数组, 将两个部分独立的排序. 快速排序和归并排序是互补的: 归并排序将数组分成两个子数组分别排序, 并将有序的子数组对并已将整个数组排序; 快速排序将数组排序的方式则是当两个子数组都有序的时候, 整个数组就是有序的. 在归并排序中, 递归调用发生在归并数组之前; 在快速排序中, 递归调用发生在切分数组之后. 在归并排序中, 一个数组被切分成两半; 在快速排序中, 切分的位置取决于数组的内容.
