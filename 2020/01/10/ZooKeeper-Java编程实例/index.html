简单的监听客户端

为向你介绍ZooKeeper的Java API, 我们开发了一个简单的监听客户端. 这个客户端监听一个Zookeeper节点的变化, 并启停一个程序作为响应.

需求

客户端有4个需求:

  • 客户端需要以下参数:
    • Zookeeper服务的地址
    • 被监听节点的名称
    • 被写入的文件的名称
    • 带参数的可执行程序
  • 客户端获取节点数据并启动可执行程序
  • 如果节点变更, 客户端将重新获取数据并重启可执行程序
  • 如果节点消失了, 客户端将关闭可执行程序

程序设计

按照惯例, Zookeeper应用分为两部分, 一个维护连接, 另一个监听数据. 在这个应用中, 名为Executor的类维护Zookeeper连接, 名为DataMonitor的类监听数据. 同样的, Executor包含主线程和可执行逻辑. 它负责用户交互. 用户传递一个可执行程序作为参数, 这个例子就会根据节点的状态关闭或重启.

Executor对象是这个应用的主容器. 它包含ZooKeeper对象, DataMonitor对象.

Watcher接口由ZooKeeper Java API定义. ZooKeeper使用它和自己的容器沟通. 它只支持一个方法, process(), ZooKeeper使用此方法沟通一系列主线程感兴趣的事件, 例如, Zookeeper连接状态或者Zookeeper会话. 本例中的Executor只是简单的将这些事件传递给DataMonitor, 让DataMonitor自己决定怎么处理这些事件. 它这么做说明了, 按照惯例, Executor或者某些类似Executor的对象”拥有”Zookeeper连接, 但是它们可以自由的将事件到事件委托给其他对象. 这被默认用来作为发送事件的通道.
